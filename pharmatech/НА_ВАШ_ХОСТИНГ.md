# PharmaTech — загрузка на ваш хостинг (StackCP)

## Что подготовлено

Папка **`pharmatech-upload`** и архив **`pharmatech-for-hosting.zip`** — готовы к загрузке.
Всё уже собрано, `npm install` и `npm run build` на хостинге не нужны.

---

## Шаги на хостинге

### 1. Проверьте Node.js

В панели StackCP найдите раздел **Node.js**, **Passenger** или **Applications**.
Если его нет — уточните в поддержке, включён ли Node.js на вашем тарифе.

### 2. Загрузите архив

Через **File Manager** или **FTP** загрузите файл:
```
pharmatech-for-hosting.zip
```
Путь: `/Users/ais/Cursor/projects/fullstack/pharmatech/pharmatech-for-hosting.zip`

### 3. Распакуйте

В File Manager нажмите на архив → **Extract** / **Распаковать**.
Должна появиться папка с файлами: `server.js`, `node_modules`, `.next`, `public` и др.

### 4. Добавьте ключи

В распакованной папке создайте файл `.env.local` (или переименуйте `.env.example` в `.env.local`).
Заполните его ключами Supabase и остальными переменными из вашего локального `.env.local`.

### 5. Запуск приложения

**Вариант А — через панель Node.js:**
- Создайте Node.js-приложение
- Укажите папку: путь к распакованной папке
- Команда запуска: `node server.js`
- Сохраните и перезапустите приложение

**Вариант Б — по SSH:**
```bash
cd путь/к/распакованной/папке
node server.js
```
(Для постоянной работы используйте `pm2 start server.js` или настройку в панели.)

---

## Важно

На хостинге должен быть установлен **Node.js** (обычно v18 или v20).
Без него приложение не запустится.

Если Node.js недоступен — уточните у хостинга, как подключить Node.js или рассмотрите VPS.
