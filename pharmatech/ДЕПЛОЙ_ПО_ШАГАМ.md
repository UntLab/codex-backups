# PharmaTech — деплой по шагам (бесплатно)

## Сколько стоит?

| Вариант | Цена | Подходит для |
|---------|------|--------------|
| **Vercel Hobby** | **Бесплатно** | Личные проекты, тест, небольшой трафик |
| **Vercel Pro** | $20/мес | Коммерция, команда |
| **Ваш StackCP** | По тарифу хостинга | Свой домен, полный контроль |
| **Supabase** | Бесплатно до лимитов | База данных (уже используете) |

**Итог:** сайт можно запустить и протестировать полностью бесплатно на Vercel.

---

## Вариант: бесплатный тест на Vercel (5 минут)

### Шаг 1. Регистрация

1. Откройте **https://vercel.com**
2. Нажмите **Sign Up**
3. Войдите через **GitHub** (удобнее) или **Email**

### Шаг 2. Создание проекта

1. Нажмите **"Add New..."** (или "+ New Project")
2. Выберите **"Deploy by uploading your project"** (или "Browse" / "Upload")
3. Если видите кнопку **"Browse"** или зону drag-and-drop:
   - Откройте Finder и перейдите в:
     ```
     /Users/ais/Cursor/projects/fullstack/pharmatech
     ```
   - Перетащите всю папку **pharmatech** в окно Vercel
4. Если просят подключить Git — можно позже; сейчас лучше **Upload**

### Шаг 3. Переменные окружения

Перед деплоем откройте раздел **Environment Variables** и добавьте:

| Name | Value (ваши данные) |
|------|----------------------|
| `NEXT_PUBLIC_SUPABASE_URL` | URL из Supabase → Settings → API |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | anon public key из Supabase |
| `SUPABASE_SERVICE_ROLE_KEY` | service_role key из Supabase |
| `NEXT_PUBLIC_APP_URL` | Оставьте пустым или укажите `https://ваш-проект.vercel.app` после первого деплоя |
| `ENCRYPTION_KEY` | 32 символа (например: `openssl rand -hex 16` в терминале) |
| `N8N_WEBHOOK_URL` | URL webhook n8n (если используете) |
| `N8N_WEBHOOK_SECRET` | Секрет webhook (если есть) |

Для каждого поля: **Add** → ввести Name и Value → **Add**.

### Шаг 4. Деплой

1. Нажмите **Deploy**
2. Подождите 1–3 минуты
3. Когда статус станет **Ready**, откройте ссылку вида `pharmatech-xxx.vercel.app`

### Шаг 5. Проверка

- Зайдите по ссылке — сайт должен открываться
- Если есть ошибки — в проекте на Vercel откройте **Deployments** → последний деплой → **View Function Logs**

## Откуда взять ключи Supabase

1. **https://supabase.com** → ваш проект
2. **Settings** (⚙️) → **API**
3. Скопируйте:
   - **Project URL** → `NEXT_PUBLIC_SUPABASE_URL`
   - **anon public** → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - **service_role** → `SUPABASE_SERVICE_ROLE_KEY` (держите в секрете)

---

## Генерация ENCRYPTION_KEY

В терминале:
```bash
openssl rand -hex 16
```
Скопируйте результат в `ENCRYPTION_KEY`.

## Лимиты бесплатного плана Vercel (Hobby)

- 100 деплоев в день
- 100 GB-hours выполнения функций в месяц
- 1 млн вызовов функций в месяц  
Для теста и небольшого сайта этого обычно хватает.
